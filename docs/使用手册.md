> **受众**：业务部门关键角色 / 产品岗 / 普通用户

## 引言

**业务全景图** 是一个团队内部使用的可视化业务流程协作工具，让产品和业务部门一起把"我们公司怎么干活的"画成一张可以多人协作、可以版本追溯、可以批注讨论的活文档。

**这本手册写给谁**：

- 产品岗 / 业务部门关键角色（你要画流程或看团队的流程）
- 普通用户（你拿到了账号，想知道怎么用）

**这本手册不写**：

- 部署 / 运维 / 故障排查
- 开发相关（API / 数据库 / 源码）

如果你需要这部分内容，请联系项目管理员或开发同学。

## 〇、先理解三种画布（重要！先看这节）

如果你是第一次用这个工具，**先花 3 分钟读完这节**——后面所有功能、按钮、提示，都建立在"三种画布"的概念之上。

### 0.1 一张图看懂

```
       你的浏览器                       服务器（内网 172.16.0.138）
    ┌──────────────────┐            ┌──────────────────────────────────┐
    │  本地草稿        │            │ 主版本画布                       │
    │ （仅你能看到）   │            │  ├─ 私有画布（仅作者 + admin）   │
    │                  │ ──保存──→  │  └─ 公共画布（所有登录用户都能看）│
    │  服务端草稿      │            │                                  │
    │ （仅你能看到，   │            │                                  │
    │  每 30s 自动存） │            │                                  │
    └──────────────────┘            └──────────────────────────────────┘
```

### 0.2 三种画布的区别

| 类型 | 谁能看 | 谁能改 | 怎么产生 |
|---|---|---|---|
| **本地草稿** | 仅你自己 | 仅你自己 | 没登录或者刚登录看到的初始空画布 |
| **服务端草稿** | 仅你自己 | 仅你自己 | 你打开服务端画布开始改后，每 30s 自动存（不影响主版本） |
| **私有画布**（private） | 作者 + admin | 作者 + admin | 你点"另存为新画布"创建出来的 |
| **公共画布**（public） | 所有登录用户 | **所有登录用户都能编辑**（详见 §0.5）| admin 把私有画布"发布"成公共 |

**一句话总结**：

- **草稿** = 你正在画，别人看不到
- **私有画布** = 你画完正式存了，但只有你和 admin 能看
- **公共画布** = admin 决定"这个流程图给所有人看"，**且让所有人一起协作编辑**

### 0.3 公共画布的"协作编辑"权限（重点）

公共画布是**多人协作画布**——任何登录用户都能加节点 / 改自己加的节点 / 加批注，但**不能改别人的、也不能物理删节点 / 删整个画布**。详细矩阵：

| 操作 | 你（普通用户）能做？ | admin 能做？ |
|---|---|---|
| 加新节点 | ✅ | ✅ |
| 改你**自己加**的节点 | ✅ | ✅ |
| 改**别人加**的节点 | ❌ 服务端会拒绝 | ✅ |
| **物理删节点**（彻底移除——含自己刚加的已保存节点） | ❌ 仅 admin（详见下方说明） | ✅ |
| **标废弃节点**（半透明保留历史，不删） | ✅ 任何人都能标 | ✅ |
| 加批注 | ✅ | ✅ |
| 关闭别人的批注 | ❌（仅批注作者可关闭） | ✅ |
| 删整个公共画布（归档） | ❌ 仅 admin | ✅ |
| 发布 / 取消发布 | ❌ 仅 admin | ✅ |

> **设计原则**：协作友好 + 历史可追溯 + 不可破坏。
> 你能加 / 改自己 / 标废弃，但不能动别人的、不能彻底抹掉痕迹——保护团队的协作历史。

> **关于"物理删自己加的节点"的特殊说明**：
>
> - 节点**保存到主版本之前**（你刚拖出来还没自动保存）→ ✅ 可以删（按 `Backspace` / 撤销，合理的"刚拖错了"场景）
> - 节点**保存到主版本之后** → ❌ 即使是你自己加的也不能再删
>
> **为什么自己加的也不能删**？
> 假设你（A）加了节点【F1】，同事（B）顺着【F1】又添加了【F2】【F3】（连线接在 F1 后）。如果你后悔删掉【F1】，B 加的【F2】【F3】就会悬空——破坏团队协作。
> **替代方案**：用【标废弃】（半透明保留 + 角标"已废弃"），后续业务流程不再走它，但历史不丢；或者请 admin 协助物理删除。

### 0.4 三条最常见困惑

> **Q：我画了好久没点保存，关了浏览器会丢吗？**
>
> A：**不会**。改完 30 秒会自动存为草稿。下次进来会问你"要不要恢复草稿"。

> **Q：我做的私有画布，别人能看到吗？**
>
> A：看不到。私有画布 = 仅你和 admin 能看。

> **Q：公共画布我能改吗？**
>
> A：能加节点 / 改自己的 / 加批注；不能改别人的、不能物理删节点。详细矩阵见 §0.3。

## 一、基础入门

### 1.1 什么是"业务全景图"

**一句话定义**：把业务流程画成可协作的可视化流程图工具。

**典型使用场景**：

- 业务调研：把现有业务流程画下来，跨部门对齐
- 流程梳理：把"应该怎么干"画清楚，作为团队共识
- 培训新人：用流程图替代大段文字，减少理解成本

**它不是什么**：

- 不是 Visio / Lucidchart 的替代品
- 定位是"团队内部业务流程的活文档"——画完就放着，可以随时回来改、随时讨论

### 1.2 登录

1. 浏览器打开 `http://172.16.0.138:3001`
2. 输入用户名 + 密码（账号找管理员开通）
3. 登录后看到的首屏：

![登录页](imgs/使用手册/01-login.png)

**首次登录**：你看到的可能是空白本地草稿 + 一个引导卡片"在右上角'另存为新画布'把当前草稿保存到服务器"——这是正常的，你还没有自己的画布。继续看 §1.4。

### 1.3 界面认识

![界面区域标注](imgs/使用手册/02-interface-overview.png)

界面分三个常驻区域 + 一个抽屉式面板：

- **① 顶栏**：菜单导航（流程图 / 管理后台 / 系统示例页面）+ 当前画布切换器 + 节点/连接数指标 + 保存按钮 + 当前用户名
- **② 左侧功能面板**：导入 / 撤销 重做 删除 / 节点工具箱（流程节点 / 判断节点 / 审批节点 / 起止节点）/ 创建分组
- **③ 中央画布**：你画图的主区域
- **④ 右侧详情面板（抽屉式）**：单击节点后从右侧滑入——显示节点的详情 / 批注；再次单击同一节点关闭

> 此外，画布**底部**有 Sheet 标签栏（一个画布可有多个工作表，类似 Excel 的 sheet 切换）。**在公共画布底部加新 sheet 分页 = 加在同一个公共画布里 = 对所有人可见**（不是新建私有画布，详见 §3.6 场景 G）。
>
> **怎么打开右侧详情面板？** 单击任何画布上的节点（流程节点 / 判断节点 / 审批节点 等），抽屉自动滑入。再次单击同一节点关闭。

### 1.4 第一次使用——按角色选路径

| 你是谁 | 推荐第一次的操作路径 |
|---|---|
| **产品 / 业务部门，刚拿到账号** | 路径 A：先点顶栏画布切换器看看团队有哪些公共画布 → 浏览熟悉工具，可以试着加批注或加节点参与协作 |
| **要画自己的流程图（不发给团队）** | 路径 B：直接在空白草稿上画 → 画完点"另存为新画布"变成你的私有画布 |
| **Admin 想发布给团队看** | 先按路径 B 画，再去 admin 后台"发布"成公共画布 |

> 公共画布是**多人协作画布**，任何登录用户都能参与编辑（细则见 §0.3 / 协作场景见 §3.6）。

#### 路径 A：浏览（或参与编辑）公共画布

1. 顶栏点画布切换器（默认显示当前画布名）
2. 下拉里能看到分组：**我的画布**（私有）/ **公共画布**
3. 点任意一个公共画布 → 进入画布
4. 你能加节点、加批注、改自己加的节点；改别人加的会被服务端拒（详见 §0.3）

![画布切换器](imgs/使用手册/14-canvas-switcher.png)

#### 路径 B：从空白开始画自己的

1. 看到的就是空白本地草稿（顶栏画布名是"本地草稿"）

![空状态引导](imgs/使用手册/03-empty-guide.png)

2. 直接拖个节点开始画（详见 §2）
3. 画到差不多了 → 点顶栏 **另存为新画布** → 输入名字 → 保存
4. 顶栏画布名变成你刚起的名字 → 这就是你的私有画布

> **画到一半要走？** 不用担心。改了 30 秒就会自动存草稿。下次进来会问你恢复不恢复。

## 二、核心功能

### 2.1 画节点

**添加节点**：左侧功能面板"节点"分类下，**点击**对应按钮（流程节点 / 判断节点 / 审批节点 / 起止节点 四类）→ 节点出现在画布中央，再拖到目标位置即可。

**打开节点详情**：**单击节点** → 右侧抽屉滑入（详情 / 批注两个 Tab）。

![添加节点 + 详情面板](imgs/使用手册/04-add-node.png)

**改节点名（两种方式）**：

- **快速改**：**双击**节点 → 节点上直接进入文字编辑（按 Enter 或点别处确认）
- **改详细信息**：**单击**节点 → 右侧抽屉打开 → 在"详情"Tab 改"名称 / 描述 / 关联节点"

**移动节点**：拖动即可。

**复制 / 粘贴节点**：`Ctrl+C` → `Ctrl+V`。注意：**粘贴出来的节点是新 ID**，跟原节点是不同的两个节点。

**删除节点**：选中 → 按 `Backspace` 键。

> **注意**：单击和双击是不同操作——单击开抽屉，双击进入节点名编辑。点错了按 Esc 取消。

### 2.2 连边

**连两个节点**：

1. 鼠标移到节点边缘 → 出现小圆点（连接桩）
2. 从小圆点拖到目标节点 → 自动连一条边

![连边操作](imgs/使用手册/05-connect-edge.png)

**改边的标签**：双击边 → 输入文字（比如"通过"/"驳回"）。

**删除边**：选中边 → 按 `Backspace` 键。

### 2.3 分组（Group）

分组用来把"逻辑相关"的节点框在一起，比如"采购流程"、"审批流程"。

**新建分组**：

1. 选中多个节点（按住 Shift 点 / 或框选）
2. 按 `Ctrl+G`（或左侧面板【创建分组】按钮）
3. 给分组起个名字

> 注意：选中的节点必须都是**你自己创建的**才能成组（公共画布上不能把别人的节点圈进自己的新分组）。

**进出分组**：把节点拖进分组框 / 拖出分组框 → 自动加入 / 移出。
也可以单击节点 → 右侧抽屉 →【移出分组】按钮（适合用键盘 / 鼠标拖不准的场景）。

**删除分组**：

- **解散分组保留子节点**：仅 admin 可操作（按 `Ctrl+Shift+G`）。普通用户想"拆开"分组：单击需要拆的节点 → 右侧抽屉 →【移出分组】，逐个移出
- **删除分组连子节点一起删**：选中分组 → `Backspace` 键（同样要满足"分组及子节点都是你创建的"权限）

### 2.4 批注（Annotation）

批注用来在节点上加讨论 / 提问 / 标记，类似"便签"。

**添加批注**：

1. **单击**一个节点 → 右侧抽屉打开
2. 抽屉顶部 Tab 切到"**批注**"
3. 在输入框写内容（最多 2000 字）→ 点【发布批注】

![批注 Tab + 数字角标](imgs/使用手册/07-annotation.png)

**节点上的批注角标**：节点上有未解决批注时，会显示一个**蓝色数字徽章**（数字 = 未解决批注数）。

**批注的状态**：

- **未解决**：抽屉里显示作者 / 时间 / 内容 + 右上角【解决】按钮
- **已解决**：变灰显示"已由 XXX 关闭于 yyyy-MM-dd HH:mm"

**谁能关闭批注**：批注作者本人 / admin（即使该画布是别人创建的）。

**批注是按节点绑定的**——节点删了，批注一起删；节点 ID 变了（比如复制粘贴产生的新 ID），批注**不会跟着**。

### 2.5 保存

**数据流转**（从你画到别人看见）：
1. 你画一笔 → 浏览器内存里改 → 顶栏显示 **● 未保存**
2. 30 秒后，自动存为"草稿"（仅你能看，不写主版本）→ 顶栏显示 **草稿已保存 14:23**
3. 你点 **保存** 按钮 → 写入"主版本"→ 别人刷新就能看到（仅他有权限的画布上）
4. 下次打开同一画布 → 如果还有未提交的草稿，弹出"草稿恢复"对话框（详见 §3.4）

画布工具栏右上角的保存按钮，会根据状态自动变化：

![画布工具栏（含保存按钮）](imgs/使用手册/08-save-states.png)

可能的状态文案（按情况显示其中一种）：

| 状态 | 含义 | 你要做什么 |
|---|---|---|
| **✓ 已保存（刚刚 / N 秒前）** | 已经存到主版本，别人能看到 | 不用管 |
| **● 未保存** | 你改了但还没保存到主版本 | 想让别人看到你的改动？点【保存】按钮 |
| **保存中…** | 服务器处理中 | 等一下，别关浏览器 |
| **草稿已保存 14:23** | 改动已自动存为草稿（仅你能看） | 30 秒一次自动存，不用管 |

> 上图展示的是"未保存"态——能看到红点 + 文字"未保存" + 蓝色【保存】按钮。点保存后会变成"已保存（刚刚）"+ 绿色对勾。

**几个重要说明**：

- 关闭浏览器前，**强烈建议手动点一次保存**，把"主版本"也更新到最新
- 点保存 = 把你的改动写到"主版本"（别人能看到）
- 不点保存 = 你的改动只在草稿里（别人看不到）

#### 另存为新画布 vs 保存的区别

| 操作 | 用途 |
|---|---|
| **保存**（已挂接画布） | 把改动写入当前画布的主版本 |
| **另存为新画布** | 创建一个**新的**画布，当前内容作为新画布的初始版本（不影响原画布） |

> **新用户第一次画完一定要点"另存为新画布"**——否则你画的东西只在本地草稿里，换台电脑或清浏览器缓存就丢了。

### 2.6 导出 / 导入

**导出**：

- **导出当前内存版本**：把你现在屏幕上看到的画布（含未保存改动）下载为 JSON
- **导出服务端版本**：从服务器拿主版本下载为 JSON（忽略你的未保存改动）

**导入**：从本地 JSON 文件还原成画布——常用于**从备份恢复**或**跨环境迁移**。

> **小贴士**：真冲突无法解决时（详见 §3.5.3），可以先"导出当前内存版本"作为备份，再做后续操作。

### 2.7 废弃节点 vs 删除

某个流程或节点不再使用了，但又想保留历史痕迹？用"标记废弃"。

| 操作 | 效果 |
|---|---|
| **标记废弃** | 节点变半透明 + 角标"已废弃" + tooltip 说明，但**不删除**（追溯历史用） |
| **删除** | 彻底从画布上移除，不可恢复（除非从历史版本恢复） |

**怎么标记废弃**：

1. 选中节点 → 右侧详情面板
2. 找到"标记废弃"开关 → 打开
3. 节点变半透明 + 角标显示

## 三、多人协作

> 团队同时编辑同一画布时如何协作。本系统采用"**保留本地改动 + 服务端尽力合并 + 真冲突时友好提示**"的设计——**几乎不会丢数据**，但你要理解几个关键提示。

### 3.1 谁能编辑这个画布？

回顾权限矩阵（详见 §0.3）：

- **私有画布**：仅作者 + admin 能看 + 改
- **公共画布**：所有登录用户都能看 + 编辑（加节点 / 改自己的 / 标废弃 / 加批注）；但**改别人的、物理删、归档整画布**这三类动作只 admin 能做

公共画布上的"用户加的节点"自动绑定 creator_id（创建人 ID），后续如果别人想改你加的节点，**服务端会以 creator_id 为依据拒绝**——保护你不会被别人覆盖改动。

![公共画布上的节点编辑视图](imgs/使用手册/13-public-canvas-readonly.png)

> **注**：上图截图取自单浏览器视角；不同用户登录后看到的实际"哪些节点能改 / 哪些不能改"是按 creator_id 区分的。

**想画基于公共画布的私人版本？** 进入公共画布会弹警告，点【📋 复制为我的私人画布】或顶栏【📋 复制为私人】按钮即可，详见 §3.6 场景 B。

### 3.2 在线状态：谁也在编辑

画布工具栏会显示一个 **👥 X 人在编辑** 角标，告诉你谁也打开了这个画布。

![画布工具栏（在线编辑者角标显示在工具栏区域）](imgs/使用手册/09-other-editors.png)

- 0 人 → 不显示
- 1-3 人 → 列出用户名
- \>3 人 → 前 3 个 + "+N"
- 鼠标悬浮 → tooltip 显示完整列表 + 最近活跃时间（XX 秒前活跃）

**重要**：

- 心跳每 30 秒上报一次，断网超过 60 秒会从列表里消失
- 看到别人在线 ≠ 你不能保存——服务器会"尽力合并"（详见 §3.5）

> **注**：上图中如果没看到"👥 X 人在编辑"，是因为截图时只有 1 个用户在编辑（角标 0 人不显示）。多人同时打开时会自动出现。

### 3.3 草稿自动保存

你改画布后，每 30 秒自动存草稿（不写主版本，只是你的临时存稿）。

- 顶栏显示 **草稿已保存 14:23**
- 草稿独立于主版本——**不会覆盖别人的改动**
- 限制：单画布草稿不超过 2MB / 全局每个用户最多 200 个草稿

> **草稿保存失败时的提示**：
>
> - "**草稿过大（超 2MB），请简化画布**" → 删除一些不必要的节点 / 边 / 详情字段
> - "**草稿数已达上限（200），请清理**" → 联系管理员或者主动保存几个画布的主版本

### 3.4 草稿恢复

重新打开有未提交草稿的画布，会弹出对话框：

![草稿恢复弹窗](imgs/使用手册/10-draft-recovery.png)

**三个选项**：

| 按钮 | 效果 |
|---|---|
| **恢复草稿** | 用草稿替换当前画布——你之前没保存的改动回来了 |
| **稍后再说** | 关闭弹窗，但草稿还在——下次打开还会弹 |
| **放弃草稿**（红色按钮） | 删除草稿，加载主版本——**不可恢复** |

> **草稿基于过期版本时的⚠️警告**：
>
> 如果你的草稿基于版本 v3，但主版本已经被别人推到 v5，对话框会显示一条警告：
>
> > 此草稿基于版本 v3，但主版本已被他人更新到 v5。如果恢复后保存，服务端会拒绝（要求基于最新版本）。建议先导出草稿备份再决定。
>
> **应对**：先点"恢复草稿"再立即"导出当前内存版本"备份成 JSON，然后决定是丢弃自己的草稿还是手工合并。

### 3.5 保存的三种结果

#### 3.5.1 直接成功（最常见）

你改 → 别人没改 → 保存成功 → 顶栏显示 **✓ 已保存 N 秒前**。

#### 3.5.2 自动合并通过（友好场景）

你改了节点 A，别人改了节点 B → 服务端自动把两边的改动合在一起 → 你看到顶栏弹出 Toast 提示：

> **已为你合并对方的改动**

不需要你做任何事，画布已经是合并后的状态。

##### 一个具体例子：A 和 B 同时加节点

假设公共画布初始是【需求 → 设计 → 开发】（v=1）。

| 时间 | 用户 A 操作 | 用户 B 操作 | 服务端版本 |
|---|---|---|---|
| t1 | 打开画布，看到 v=1 | 打开画布，看到 v=1 | v=1 |
| t2 | 加【测试】节点（连在【开发】后） | 加【部署】节点（连在【开发】后） | v=1 |
| t3 | 点【保存】 | （还在改） | **v=2**（A 的版本：含【测试】）|
| t4 | （已保存） | 点【保存】 | **v=3**（自动合并：含【测试】+【部署】）|

**最终结果**：公共画布 v=3 上**两人加的节点都在**——【需求 → 设计 → 开发 →【测试】+【部署】】。

**B 浏览器的反应**：

- 点保存前，B 屏幕上**只看到自己加的【部署】**（因为 B 没刷新，看不到 A 加的【测试】）
- 点保存后，顶栏弹 Toast"**已为你合并对方的改动**"，画布上**自动出现【测试】节点**

> 这就是"自动合并"的魔法——只要你们加的是不同的东西，系统帮你合并；不需要约时间错峰开。

**自动合并能 cover 哪些操作**：

| 你的动作 | 对方的动作 | 自动合并？ |
|---|---|---|
| 加节点 X | 加节点 Y | ✅ |
| 改节点 X 的名字 | 改节点 Y 的描述 | ✅ |
| 改节点 X 的名字 | 改节点 X 的位置（不同字段） | ✅ |
| 加批注 | 改节点（不冲突） | ✅ |
| 加节点 | 标废弃别的节点 | ✅ |
| **改节点 X 的名字** | **改节点 X 的名字（同字段）** | ❌ 真冲突，弹窗见 §3.5.3 |

#### 3.5.3 真冲突弹窗（需要你决定）

你和别人改了**同一个节点的同一个字段**（比如都改了节点 A 的名字）→ 服务端无法判断要谁的 → 弹出对话框：

![真冲突弹窗](imgs/使用手册/11-conflict-dialog.png)

**对话框显示**：

- 对方的版本号（**v?**）
- 冲突明细列表：每条显示节点 ID / 字段名 / 说明

**两个选项**：

| 按钮 | 效果 | 推荐场景 |
|---|---|---|
| **保存草稿并查看最新版本**（蓝色主按钮） | 把你的改动存为草稿 + 重载对方最新版本 → 你可以看到对方改了什么，再决定如何手工合并 | **多数情况推荐用这个**——零丢失 |
| **继续编辑草稿** | 保留你的改动，不重载——下次保存时服务端会再尝试合并（可能再次冲突） | 你确定要坚持自己的版本时用 |

> **绝对不要**：
> - 看到冲突弹窗就 F5 / 关浏览器（草稿可能还没存，会丢改动）
> - 慌乱中乱点——先看清楚冲突的是哪个节点的哪个字段

> **罕见场景：基础版本过期**
>
> 如果你打开画布很久没动 + 别人改了很多次 → 点保存可能弹"**画布历史版本已超出可合并范围**"——服务器清理了你打开时的旧版本快照，无法做合并了。**两个按钮**：「**保存草稿后重载**」（推荐）会先存草稿再重载最新版本；「**留在草稿**」不重载，自动保存暂停。**怎么避免**：长时间不动的画布，每隔半小时点一次保存或刷新页面。这个对话框在 5 人内网协作场景下很少出现。

### 3.6 协作工作流（按场景速查）

不同场景下的操作路径——找到你的场景，照着做。

#### 场景 A：我想直接参与编辑团队的公共画布

1. 顶栏画布切换器 → 选择目标公共画布
2. **首次进入公共画布会弹警告**：
   - ⚠️ "你正在打开公共画布「xxx」"
   - 三个选项：
     - 📋【复制为我的私人画布】→ 跳到场景 B（推荐先在私有副本上排练）
     - 👀【只查看公共画布】→ 关弹窗保持只读模式（看完想改可以再点顶栏「📋 复制为私人」）
     - 【继续编辑公共画布】→ **必须勾选确认复选框**才能点击；勾选后修改会立即对所有人可见
3. 选「继续编辑公共画布」后：直接加节点 / 加批注 / 改你自己加的节点 → 点【保存】（同会话内同一画布只弹一次警告）

> 你能做什么 / 不能做什么见 §0.3 权限矩阵。**核心红线**：不能改别人加的节点、不能物理删节点（想"撤销"用【标废弃】，见场景 E）。
>
> 为什么进画布要弹警告？公共画布上的修改**自动保存且对所有人可见**，节点加上后**普通用户不能再删**。系统建议你先复制到私人副本上排练（场景 B）。

#### 场景 B：我想基于公共画布画自己的版本（不影响公共版）

**两种入口都可以：**

**入口 1**（推荐，加载即引导）：进入公共画布弹警告时，点【📋 复制为我的私人画布】→ 起个名字（默认「原名 - 副本」）→ URL 自动跳到新私有画布

**入口 2**（已经在画布里）：顶栏点【📋 复制为私人】按钮（只在普通用户 + 公共画布 + 已挂接服务端时显示）→ 起个名字 → 跳新画布

复制后：
- 新画布是你的【私有画布】，可以**自由编辑所有节点**（包括从公共版复制过来的别人节点——因为是你新画布的副本，creator_id 全归你）
- 你的改动**不影响原公共画布**
- 如果你在公共画布上点复制时已经有未保存改动，那些改动也会一起带到副本里（公共画布不会因此被保存）

> 适合场景：调研后想画"现状 vs 目标"两版对比；探索一个改进方案但不想干扰团队主流程；或者只是想在私有空间试探性地改一改。

#### 场景 C：我私有副本改完了，想让团队都能看（"提交合并"）

**当前应用没有"自动合并到公共画布"按钮**。两条路：

| 方式 | 适合什么时候用 |
|---|---|
| **C1 直接在公共画布上加你的节点**（推荐） | 多数情况下走这条。公共画布本来就是多人协作的，你新加的节点对所有人可见，且自动绑定 creator_id 保护你不被覆盖 |
| **C2 找 admin 帮忙发布** | 你画的是"另一个流程图"（不是补充已有的，而是独立的新一份）→ 把你私有画布发给 admin，由 admin 在后台【发布】成新的公共画布 |

> 简单说：**改公共画布上的某个流程**直接在公共画布上加节点（场景 A 路径）；**画一个全新的流程**画完后请 admin 发布（场景 F 路径）。私有副本主要用于"试探性探索 / 个人草稿"（场景 B），改完不一定都要发布。

#### 场景 D：我和别人同时改公共画布（A 加 X 节点，B 加 Y 节点）

详见 §3.5.2 的具体例子。简单说：**只要你和别人加的节点不重叠**（节点 ID 不冲突，没改同一个字段），系统会**自动合并**——你保存时顶栏弹"已为你合并对方的改动"Toast，画布上会**自动出现别人加的节点**。

#### 场景 E：我想"撤销"一个节点（不删，但不让它生效）

公共画布上**已保存的节点不能物理删**——即使是你自己加的（保留协作历史，避免别人接在你节点上画的内容悬空）。三种替代方式：

- **标废弃**（推荐）：选中节点 → 右侧抽屉【标记为已废弃】 → 节点变半透明 + 角标，后续业务不走它，但历史记录保留
- **加批注说明**：节点上加批注"此节点已弃用，请改用 XXX"
- **请 admin 物理删**：admin 在公共画布上有完整删除权限，但 admin 删之前会权衡"是否会让别人加的节点悬空"

> **特例**：节点刚加出来**还没自动保存**（30 秒内）的话可以按 `Backspace` 直接删——这种情况是"刚加错了/刚画错了"，不会破坏协作历史。
>
> **如果你点了删除却看到错误提示**："无法删除——公共画布上的节点只允许管理员物理删除"，是正常的；系统会自动撤销前端的删除操作（节点在 1-2 秒内会重新出现），数据安全无影响。

#### 场景 F：admin 想发布私有画布为公共

1. admin 进入 admin 后台 → 共有画布管理 tab
2. 找到目标画布 → 点【发布】按钮
3. 此时画布从 private 变 public，所有登录用户都能看到 + 协作编辑

> 取消发布同款路径——【取消发布】按钮把 public 退回 private（仅作者 + admin 可见）。

#### 场景 G：我在公共画布底部点了【+ 画布】加新 sheet 分页，这个分页归谁？

**仍然归这张公共画布**，对所有人可见。

**关键概念区分**：
- **顶栏【📁 切换画布】= 切 canvas**（数据库里独立的一张图，有自己的 public/private 属性）
- **底部【+ 画布】= 加 sheet 分页**（同一个 canvas 内的多个工作表 tab，类似 Excel 的多 sheet；**共享 canvas 的可见性**）

所以在公共画布底部加的"画布 2 / 画布 3..."分页，跟你加节点 / 改字段一样，**自动保存且对所有人可见**。如果你想做"个人草稿不让别人看见"的实验，请走**场景 B**（顶栏【📋 复制为我的私人画布】先复制到自己空间，再在副本上随便加 sheet）。

> 想删自己加错的 sheet？右键该 sheet tab → 删除（仅 sheet 上无节点、或你是该 canvas 的 owner / admin 时可用）。

### 3.7 协作的几条纪律

- ✅ 大改之前先在协作群里说一声："我开始改 XX 画布的 XX 部分"
- ✅ 看到 **👥 X 人在编辑** 时，避免大范围拖动 / 删除其他人正在动的区域
- ✅ 真冲突弹窗出现时，先深呼吸——这不是 bug，是协作正常现象
- ✅ 不确定时点 **保存草稿并查看最新版本**——零丢失，最安全
- ❌ 不要在冲突弹窗弹出时 F5 / 关浏览器
- ❌ 公共画布的发布请走 admin 流程，不要直接改

### B. 名词速查

| 术语 | 含义 |
|---|---|
| **画布**（Canvas） | 一个完整的业务流程图（可以包含多个工作表）；有 public/private 可见性属性 |
| **工作表**（Sheet） | 画布里的一页（类似 Excel 的 sheet）；**共享所属 canvas 的可见性**——公共画布上的所有 sheet 都对所有人可见 |
| **节点**（Node） | 流程中的一个步骤 / 实体 |
| **边**（Edge） | 节点之间的连线 |
| **分组**（Group） | 把多个节点框在一起的逻辑分组 |
| **批注**（Annotation） | 节点上的讨论 / 提问 / 标记 |
| **草稿**（Draft） | 自动保存的临时存稿，仅你能看 |
| **主版本** | 服务器上正式存档的版本，别人能看到 |
| **基础版本**（Base Version） | 你打开画布时的版本号——保存时服务端基于它判断有没有冲突 |
| **真冲突**（Real Conflict） | 双方改了同一个节点的同一个字段，无法自动合并 |
| **自动合并**（Auto Merge） | 双方改了不同节点 / 不同字段，服务端自动合并 |

- **系统故障 / 部署问题**：运维 / 开发同学

具体联系人请向你的项目负责人确认。
