**Low 1（建议）：onAnnotationBadgeClick nodes 闭包**

[src/components/FlowCanvas/index.tsx onAnnotationBadgeClick] 依赖 nodes 闭包查找 targetNode。正常点击来自当前渲染节点，闭包通常正确；但 nodes 频繁变化时 callback/value 频繁重建，极端快速更新下可能用到上一帧 targetNode.data。

**修法**：可保留进入 P3E-4；若想更稳，可用 useRef 同步保存最新 nodes，让 onAnnotationBadgeClick 依赖 [setNodes, requestPanelTab] 从 ref 读节点。

**Low 2（建议）：Provider value 抖动**

[src/components/FlowCanvas/index.tsx annotationBadgeContextValue] Provider value 随 nodes 和 annotationsBundle 引用变化重建 → CustomNode/GroupNode 跟随重渲染。当前内网 < 1000 节点规模不阻塞。

**修法**：无需必修。后续画布操作卡顿再改为稳定 callback。

**Low 3（通过）：pendingPanelTab effect 策略**

nodeId 不匹配时不清零策略可接受。deps 包含 pendingPanelTab + selectedElement + onPendingPanelTabConsumed，后续 selectedElement 更新会再次执行 effect。选中 edge 或 null 时清零处理 stale pending。

**Low 4（通过）：getCollapsedContent**

按行数和字符数双条件折叠的实现覆盖了少字多行、长单行、多行长文本和边界 200 字/6 行场景。

**Low 5（通过）：ErrorBanner**

onDismiss 改 optional 后，fetchError 场景不再显示无效关闭按钮；submit/mutation error 仍可关闭。

**Low 6（通过）：fetchSeqRef**

fetchSeqRef 在 disabled 判断前递增，旧 GET 的 finally 不应再影响当前 key 的 loading；一审该项可视为误读或已由现代码满足。
