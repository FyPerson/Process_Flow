**High 1：徽章点击未显式选中节点**

[src/components/CustomNode/index.tsx:285, src/components/GroupNode/index.tsx:343, src/pages/BusinessFlowVisualization/index.tsx:117](E:/业务全景图/src/components/CustomNode/index.tsx) 徽章 onClick 调用了 stopPropagation，但 onBadgeClick 只设置 pendingPanelTab，没有显式选中对应节点，也没有打开 NodeDetailPanel。点击未选中节点的徽章时 selectedElement 不会变成该节点，NodeDetailPanel 的 pendingPanelTab effect 因 nodeId 不匹配不会消费；面板关闭时也不会自动打开。

**修法（codex 推荐）**：把徽章点击处理下沉到 FlowCanvas 内部，Context value 提供 onBadgeClick(nodeId)，在其中用 setNodes/setEdges 显式设置选中状态、setSelectedElement 为目标节点、setShowPanel(true)，再设置 pendingPanelTab。若 pendingPanelTab 必须留在 BFV，可给 annotationsBundle 增加 requestPanelTab(nodeId, tab) 并由 FlowCanvas 包装 selection + request。

**Medium 1：pendingPanelTab 过期不清理**

[src/components/NodeDetailPanel/index.tsx:197](E:/业务全景图/src/components/NodeDetailPanel/index.tsx) pendingPanelTab 在 selectedElement 不匹配、当前选中 edge、或目标节点已不存在时不会被消费也不会清零。之后如果用户再次选中同 nodeId 的节点，可能消费一个过期的 pending 请求，出现非用户当前意图的 tab 跳转。

**修法**：FlowCanvas 显式选中修复后风险降低；仍建议给 pending 请求加 generation/timestamp，或在 NodeDetailPanel effect 中当 pending 目标节点不存在、选择了 edge、或用户手动切换其他节点时调用 onPendingPanelTabConsumed 清理。

**Medium 2：useAnnotations disabled 分支不递增 fetchSeqRef**

[src/hooks/useAnnotations.ts:150](E:/业务全景图/src/hooks/useAnnotations.ts) 切到 disabled key 时 performFetch 只 setLoading(false)，但没有递增 fetchSeqRef。旧 GET 回包的 finally 如果序号仍是最新，可能把 disabled 期间的 loading/error 状态写回；虽然 gen 校验会阻止落地 annotations，但 loading 语义仍依赖旧请求时序。

**修法**：currentKey 变化时递增 fetchSeqRef 或在 disabled 分支也 `++fetchSeqRef.current`，确保旧 GET 的 finally 不再影响新 key 的 loading。

**注**：这是 P3E-2 三审 medium 1 修法的小遗漏（P3E-2 三审 medium 1 加了 setLoading(false) 但忘了 ++fetchSeqRef）。

**Medium 3：长批注折叠只按字符**

[src/components/NodeDetailPanel/AnnotationPanel.tsx:310](E:/业务全景图/src/components/NodeDetailPanel/AnnotationPanel.tsx) 长批注折叠只按 `annotation.content.slice(0, 200)` 截断。若内容少于 200 字但超过 6 行，displayContent 实际仍显示全文再追加省略号，没有真正按 6 行折叠。

**修法**：当 lines.length > 6 时优先使用 `lines.slice(0, 6).join('\n') + '…'`；当字符数超过 200 时再按字符截断，二者取更短结果。

**Medium 4：ErrorBanner fetchError 场景关闭按钮无效**

[src/components/NodeDetailPanel/AnnotationPanel.tsx:246](E:/业务全景图/src/components/NodeDetailPanel/AnnotationPanel.tsx) fetchError 的 ErrorBanner 传入空 onDismiss，但 ErrorBanner 仍渲染关闭按钮。用户点击关闭没有效果，会让 inline 错误条看起来坏掉。

**修法**：给 ErrorBanner 增加 `dismissible?: boolean`，fetchError 场景隐藏关闭按钮；或者在组件本地加 dismissedFetchError 状态，直到 fetchError 对象变化再显示。

**Low 1：折叠分支冗余**

[src/components/NodeDetailPanel/AnnotationPanel.tsx:315](E:/业务全景图/src/components/NodeDetailPanel/AnnotationPanel.tsx) `annotation.content.slice(0, 200) + (lines.length > 6 ? '…' : '…')` 两个分支完全相同，降低可读性，也掩盖了按行折叠未实现的问题。

**修法**：改成清晰的 getCollapsedContent(content) helper（与 medium 3 修法一致）。

**Low 2 通过：effect 顺序成立但脆弱**

切节点重置 details 与 pendingPanelTab 消费目前依赖 React effect 声明顺序，当前 reset effect 在前、pending effect 在后 —— 徽章触发时最终应落到 annotations。结论成立但维护上较脆。

**修法**：可保留；若要降低维护风险，可合并成一个 effect。

**Low 3 通过：多 sheet 隔离正确**

annotationKey(sheetId, nodeId) 复合 key + processedNodes 按 activeSheetId 派生 + AnnotationPanel create 传 sheetId+nodeId 整体不易串 sheet。

**Low 4 通过：Provider 位置正确**

Provider 放在 FlowCanvas 外层、ReactFlowProvider 内部由 FlowCanvas 创建，不会阻断普通 React Context 传递。但**修法时建议把 Provider 移到 FlowCanvas 内部**（Context value 需要 reactFlowInstance setNodes / setSelectedElement 闭包）。
