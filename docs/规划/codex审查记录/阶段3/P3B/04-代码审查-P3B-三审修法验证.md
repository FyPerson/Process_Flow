## 第二层（codex 原话技术细节）

### 🟡 Issue 1（medium / contradiction）— 半组选区文档与实现不一致

- **Location**：`src/hooks/useFlowClipboard.ts / remapClipboardNodes 行为约定 b`；`src/hooks/useFlowClipboard.test.ts case 3 标题`
- **Problem**：半组选区的行为说明仍写着"position 保持原值"，但实现和断言实际是 parentId 未命中时按无父节点处理并加 OFFSET。该矛盾正好落在三审 Issue 2 的修法边界上，未来开发者可能再次按"保持原值"或"转绝对坐标"改出不一致逻辑。
- **Suggestion**：把 useFlowClipboard.ts 行为约定 b 改成"删 parentId/extent；position 加 OFFSET，按无父孤儿降级处理；不做绝对坐标转换"。同时把 case 3 标题改为"position 加 OFFSET（不支持转绝对）"，不要再出现"保持原值"。

### ⚪ Issue 2（low / edge-case）— lint 行内注释剥离的字符串边界

- **Location**：`scripts/check-id-generation.mjs / 行内注释剥离`
- **Problem**：当前用 rawLine.indexOf('//') 粗略剥离行内注释，若真实违规代码出现在含有 URL 字符串之后的同一行，例如 `const u='http://x'; const id='n_'+Date.now()`，扫描内容会在 http:// 的 // 处被截断，导致漏报。内网项目里这种写法概率低，但这是明确的绕过边界。
- **Suggestion**：短期可接受；若要补强，改为只跳过 trimStart 以 // 开头的整行注释，不剥离行内注释，或者实现一个轻量 lexer 跳过字符串字面量后再识别 // 注释。更简单的折中是增加自检 case 覆盖"URL 字符串后仍有违规 ID 拼接"。

### ⚪ Issue 3（low / edge-case）— lint 字符串拼接的更深绕过

- **Location**：`scripts/check-id-generation.mjs / FORBIDDEN regex`
- **Problem**：字符串拼接拦截只覆盖 `'<前缀>_' + 时间戳 token` 的顺序，仍可能被 `'<前缀>_' + String(Date.now())`、``'<前缀>_' + `${Date.now()}` ``、`Date.now() + '_n'`、或先把 Date.now() 存入非 ts/now/timestamp/baseTs 变量再拼接绕过。作为回归保护脚本可以接受，但不应被理解为完整静态分析。
- **Suggestion**：如果门禁目标是防常见误用，当前强度足够。若要进一步提高，可加入 `String\(Date\.now\)`、`Number\(new Date\)`、变量名 idSeed/timeSeed 等 token，或将 ID 生成收敛到 newNodeId/newGroupId/newEdgeId 的导入使用检查。
