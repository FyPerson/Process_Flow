## 三、踩坑与新纪律

本阶段 3 个新踩坑，1 条值得入 auto memory（其他归 session 日志）：

- **React Flow 子节点 position 是相对父坐标** — 复制粘贴时不能统一加 OFFSET，父被一起复制时 child 必须保持相对坐标不变。该认知陷阱**未来同类批量平移路径都要警惕**（P5 合并算法导入节点必触）。已在 [`useFlowClipboard.test.ts` case 2](../../../../src/hooks/useFlowClipboard.test.ts) 单测层固化锁住，触发时间记入 P3B/03-二审修法验证 Issue 1。
- "修法描述 ≠ 修法实现"踩坑（codex 二/三审连续 2 轮揪出 `idMap` 写没用 + 死代码分支）→ **未升级 auto memory**（仅 1 个阶段案例，session_20260506 记为临时偏好。如未来再出现 ≥2 次同类，升级到 `feedback_ai_collaboration.md` "边界" 段）。
- IDE 红波浪线（tsconfig.client.json exclude *.test.ts 副作用）→ 修法已固化为"加新 tsconfig 子项目时建 references"约定写进 [CLAUDE.md](../../../../CLAUDE.md) 工程基建段。
