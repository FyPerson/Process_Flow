#### #1 readability / canvases.publish.test.ts: 测试 #12 + 部分字段升级用例

**problem**：测试中仍有手写迁移回填 SQL；其中 #12 还保留旧语义 `WHERE published_at IS NULL` 和直接赋值，不等同于当前 0003 的"任一字段缺失 + COALESCE 分字段补"。另一个"真升级"部分字段用例其实是在跑完 0003 后插入数据再手动执行复制 SQL，名称容易让维护者误以为它执行了 migration 文件本身。

**suggestion**：删除已被 dedicated migration 测试覆盖的 #12，或把手写 SQL 更新为当前 0003 的完整 COALESCE 版本；部分字段用例建议改名为"回填 SQL 语义：部分字段缺失仅补缺失值"，并在注释里明确它不是旧库真实升级路径。

#### #2 edge-case / canvases.publish.test.ts: dedicated migration 升级测试后 3 个用例

**problem**：后 3 个 migration 用例用 `files[0]`、`files[1]`、`files[2]` 选择迁移文件，而第一个用例用 `startsWith('0001_')` 等显式查找。当前能工作，但未来若插入更早编号或调整文件排序，测试可能跑错迁移文件。

**suggestion**：复用第一个用例的显式查找方式，在每个用例或共享 helper 中按 `startsWith('0001_')`、`startsWith('0002_')`、`startsWith('0003_')` 定位文件，避免依赖数组下标。
