## 你的推荐方案
1. 取舍 1：A。私有画布必须带 `Authorization`，fetch+blob 最简单；记得读 `Content-Disposition` 文件名。
2. 取舍 2：更推荐 D：顶栏“画布操作/更多”菜单。导出是只读能力，readOnly 用户也应可见，不该绑在左侧编辑工具栏。
3. 取舍 3：A。导入会创建并跳转新画布，放编辑入口里够清晰；必须登录，不在 readOnly 态出现。
4. 取舍 4：window.alert 够用。P2I 快糙好阶段没必要做 modal；Zod issue 拼成可读路径即可。
5. 取舍 5：A。冲突态直接给“导出本地副本”最符合逃生口语义，不能混进服务端导出。

## 关键风险
- 客户端最好先拦截文件大小，避免等 413。
- 导出 blob 下载会丢后端文件名，需解析 header 或 fallback。
- 导入成功后要清掉当前未保存/冲突心智，明确跳到新 `canvasId`。
- 本地副本导出必须用当前内存 project，并在文件名标注 local/conflict。
