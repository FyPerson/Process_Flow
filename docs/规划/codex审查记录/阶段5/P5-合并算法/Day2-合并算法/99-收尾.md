## 关键架构决策（不重复 02-拍板 / 03-拍板 / 10 已写过的细节）

1. **D5 helper 三件套**：`validateDeltaBPermissions / rewriteNodesFromMeta / syncNodesMetaFromDeltaB`（阶段 A 抽取，阶段 C 复用）— 快速路径与合并路径副作用代码完全一致，避免双套漂移
2. **detector vs applyDelta 分层职责**：detector 只看 delta（base→A、base→B 比对），applyDelta 看 mergedData 完整性兜底；DataIntegrityError 透传给 saveCanvas → route 层映射 500 + 'data_integrity_error'
3. **deltaB 单一信息源**（codex 阶段 C 取舍审 high 风险吸收）：saveCanvas 自己 computeDelta(base, incoming) 拿 deltaB 给 helper 用；tryMerge 内部也 computeDelta 一次但不外传，避免 G4 漂移
4. **userA 来源 = canvas_versions.saved_by**（codex 阶段 C 取舍审戳穿）：不能用 canvases.updated_by — patchCanvasMeta/publishCanvas/archiveCanvas/unpublishCanvas 会写主表 updated_by 但不写 canvas_versions
5. **权限校验先于合并冲突**（codex 阶段 C 取舍审 medium 风险吸收）：computeDelta → validateDeltaBPermissions → tryMerge → rewrite → UPDATE → INSERT → syncMeta；403 越权先于 409 合并冲突返回
