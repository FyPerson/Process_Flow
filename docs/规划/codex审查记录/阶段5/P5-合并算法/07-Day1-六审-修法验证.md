## Issues 按 severity 排列

### High（1 条 —— 时机被动调整：审计提前）

#### H1 [edge-case] 旧数据审计应提前到 Day 2 接入前（不能挂 Day 4）

**位置**：`server/schemas/canvas.ts: superRefine` + `server/services/merge/computeDelta.ts: assertProjectIntegrity` + saveCanvas 合并接入点

**问题**：新增 parentId 硬约束会立即影响**所有 PUT/POST body validation**；如果生产 canvases.data 或 canvas_versions.data 历史快照里存在 group 嵌套、自引用或 parentId 指向非 group，**客户端即使不修改这些节点也可能在保存时被 schema 拒绝**。Day 2 接入 computeDelta 后，baseSnapshot/currentData/clientData 任一历史脏数据也会触发 DataIntegrityError，若未专门捕获会表现为 **500**，而不是可恢复的 reload/conflict 流程。

**建议**：Day 2 接入前至少做一次**只读审计脚本**，扫描 canvases.data 与 canvas_versions.data 的 parentId 违规项；若有数据，先执行清理迁移或在合并路径把 baseSnapshot 的已知历史完整性错误映射为 `base_version_expired/reload`。**部署前不要只挂账到 Day 4，因为 schema 已经改变请求兼容性。**

→ **本判断与 v5 M3 同主题但时机变化**：v5 codex 说"Day 4 部署前"，v6 codex 升级为"Day 2 接入前"，因为 schema 已改使**当前的 PUT 请求**可能立即受影响（不是只在合并路径）

### Medium（1 条）

#### M1 [missing-constraint] CONNECTOR_DIFF_FIELDS 没等价静态保护

**位置**：`server/services/merge/types.ts: CONNECTOR_DIFF_FIELDS / StorageConnector`

**问题**：NODE_DIFF_FIELDS 已有 AssertNever 双向闭环，但 CONNECTOR_DIFF_FIELDS 仍没有等价静态保护。Day 2 的 edge_semantic_conflict 需要比较所有非语义 connector 字段，若后续 FlowConnector/EdgeSchema 新增字段但 CONNECTOR_DIFF_FIELDS 漏加，会出现静默自动合并或漏报冲突。

**建议**：在 Day 2 detector 前补一个不带索引签名的 `StorageConnectorContentFields` 或 ConnectorDiffFieldUnion，并对 CONNECTOR_DIFF_FIELDS 做 AssertNever 双向断言。SHEET_META_DIFF_FIELDS / PROJECT_META_DIFF_FIELDS **不建议照搬**双向断言，因为它们是有意选择的元信息子集，应改为用注释或显式 MetaFields 类型表达边界。

→ **Day 2 前 15 分钟修**：抄 NODE 的写法套到 CONNECTOR

### Low（2 条）

#### L1 [ambiguity] parentId 指向 is_deprecated=true 的 group 业务语义未拍板

**位置**：`server/schemas/canvas.ts: parentId 校验` + `server/services/merge/computeDelta.ts: assertProjectIntegrity`

**问题**：parentId 指向 `is_deprecated=true` 的 group 当前会通过校验。若业务语义认为"废弃分组仍存在但仅标记状态"，这是合理的；若前端会隐藏或弱化废弃分组，则可能产生非废弃子节点挂在废弃分组下的可见性/编辑语义问题。

**建议**：在 Day 2 detect/apply 前明确拍板。若不可，应在 schema/assertProjectIntegrity 或 detector 中拒绝/冲突化；若可，应在类型注释或方案中写明。

→ **Claude 判断**：本项目 P3C 是"标废弃只是标记状态，节点仍存在"——废弃 group 仍可作为 parent 是合理的。Day 2 加注释说明即可，无需校验。

## Risks（codex 列）

1. 历史数据不满足新 parentId 约束时，现有客户端保存旧画布可能被 400/校验错误阻断
2. Day 2 合并路径直接调用 computeDelta 后，历史 baseSnapshot 脏数据可能从可恢复并发冲突升级为 500
3. connector 字段白名单无静态闭环，后续新增边字段时可能漏入 diff/冲突检测
4. 废弃 group 是否能继续作为 parent 未拍板，可能在 UI、权限和合并语义之间产生不一致
