### #2 autosave effect 在 dirty + saving=false + autoSaveDisabled=false 下重新触发

`useMultiCanvas line 1230-1246` deps = `[dirty, canvasId, saving, autoSaveDisabled, project]`：

- B 方案 changeSeq 不等时：dirty 保持 true（已是 true 因为有后续编辑） ✅

- merged 路径 line 996 setAutoSaveDisabled(false) ✅
- project 因后续编辑变化 → useEffect 重新触发 → 5s 防抖延后到下个 tick

**确认**：autosave 会再次触发 → 用旧 serverVersionRef 作 baseVersion → server 进合并路径 → tryMerge(base_v=mergedFromVersion, current=server mergedData, incoming=本地后续编辑) → 把 A 改动 + B 后续编辑全部合并 → mergedData 含 A+B+B 后续。

### #3 B 方案的"延后看到 A"边界场景

用户连续编辑 → 多轮 save 都 changeSeq 不等 → 永远在保存自己的进度但看不到 A 改动 → 服务端 conflict_logs 会记多条 auto_merged。

5 人内网下：
- "用户编辑速度持续 > 服务端 save 速度"实际不太可能（人手编辑 vs 局域网 < 1s 网络）
- 即使发生，后果是用户体验"我看不到对方改动"延后，**不丢数据**
- 建议归 Day 4 用户文档：合并 toast 文案"已合并 A 的 X 条改动"——让用户感知到何时合并完成
