## 关键架构决策

1. **B 方案不变量从运行期约定升级为类型不变量**（切片 1+2）：`mergeSavePlan` 纯函数的 `DeferMergedPlan` 类型上不携带任何"推进 ref/替换 project"字段，编译期堵 B 方案绕过路径。三层守门：类型层 + 单测 g1/g2 + grep `check:invariants`
2. **F-16 (f)/(h) 不引入 RTL/jsdom**（04 范围判读审 codex 拍板）：抽 `saveErrorDispatcher` 纯函数 + node --test 跑分支决策；类型层 `shouldDeleteDraft: false` 字面常量守门 (h)；不接受注释/grep 单独关闭债务（codex H2/M1 戳穿）
3. **F-14b 改"单浏览器 + API 模拟对方推送"模式**（06 设计取舍审 codex 拍板）：避开双方 autosave 5s 防抖竞态；F-14a 已证双 ctx 基建可用 + 服务端 source-of-truth；F-14b 聚焦"真实浏览器保存路径触发后的 Toast / ConflictResolutionDialog UI 响应"
4. **C 修法：catch 块 setServerError 挪到 rethrow 分支**（F-14b 揪出 BFV 真 bug 后修）：BFV [887] 错误兜底页判 `serverError || !project` 退化整个画布；conflict / base_version_expired 路径误触发让 ConflictResolutionDialog 永远渲染不出来。修法把 setServerError 仅在真致命错误（rethrow path）才写
5. **L2 修法：switch + never 穷尽检查**（05 小范围审）：catch 块 `switch (errorPlan.action)` + `default: never`；反向验证临时加新 action 让 tsc 真报错验证守门生效
6. **R5 修法：补 currentVersion 类型非法测试**（05 小范围审）：旧测试只测"缺字段"，没测"字段类型非法（string/null）"；新增 case 9-10 覆盖
7. **M1 修法：planSaveError 入参 unknown + 形状守卫**（07 末尾审）：防 catch 接到非 ApiError throw（null/undefined/普通 Error/字符串/数字）二次异常；非 ApiError 形状统一返回 rethrow plan；测试 +6 case (11-16) 反向验证 6 个 rethrow case 全过
