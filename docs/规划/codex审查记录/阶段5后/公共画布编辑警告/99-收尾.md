# 99 收尾 — 公共画布编辑警告（v1.18.0，2026-05-09）

> 阶段 5 后 fix 链 v1.18.0；不属于阶段编号。

## 背景

普通用户载入公共画布时存在双重风险：

1. **autosave 5s 防抖**会把试探编辑直接 PUT 到服务端 → 公共画布对所有人可见的内容被污染
2. **公共画布上普通用户不能删节点**（v1.16.2 / canDeleteNodeData 强制：public 画布上 only admin can delete saved nodes）→ 误改后无法撤回

用户诉求：在普通用户首次"试探性编辑"公共画布时**拦截 + 引导先复制为私人副本**。

## 落地方案（C 方案：弹窗 + 主动按钮）

- **被动**：普通用户加载未 ack 公共画布 → 加载即弹拦截窗（3 选项 + 复选框二次确认）
- **主动**：SaveStatus 顶栏挂「📋 复制为私人」按钮（普通用户 + 公共画布 + 已挂接服务端时显示）
- 两个入口共用 `handleSaveAsCopy`：弹 prompt 命名 → `createOnServer({ visibility: 'private' })` → URL 跳新画布
