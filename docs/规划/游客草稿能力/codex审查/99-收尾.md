## 四、关键决策记录

### 1. codex 审查归档嵌套式（5/13 拍板）
- 新功能用 `docs/规划/<功能名>/{方案.md, codex审查/}` 容器
- 历史阶段 2-5 保留原结构 `codex审查记录/阶段X/PXX/`
- 详见 [codex审查记录/README.md](../../codex审查记录/README.md)「两种归档风格的边界」段

### 2. 裸 React Flow（方案 §3.2 / 5/13 拍板）
- 不复用 FlowCanvas / useFlowOperations / CustomNode（避免污染主组件）
- DraftSandbox 独立壳，未来改造不影响主应用
- 反向断言守门（D-6 + D-8 M4）防未来重构破坏

### 3. 仅 PNG 导出（5/13 + D-8 M1 拍板）
- MVP 只暴露 PNG UI，`exportCanvasAsJpg/Svg` 工具函数保留备用
- 理由：PNG 已覆盖 90% 场景 / Tree shaking 共享 html-to-image 包 / 真有需求再开

### 4. 不持久化 viewport（5/14 拍板）
- 符合"临时工具"定位，刷新 fitView 重置视角
- 避免缩到一片空白找不回的尴尬

### 5. 不支持多 sheet（5/14 评估后维持方案 §2.2 决策）
- 简化产品定位
- 复杂需求建议办账号画正式画布

### 6. 路由完全独立（方案 §3.1 / D-1 落地）
- `/draft` 提到 AuthProvider 外，App.tsx 顶层 Routes 拦截
- 未登录用户也能直接访问

### 7. 撤销栈用 Context callback 而非 window CustomEvent（D-8 H2+M5 修法）
- 避免全局事件串扰 + 异步竞态
- `DraftNodeRenameContext` 注入 handleRename，DraftSandbox 在 setNodes updater 同步合成 next + 推快照

### 8. 删除节点同步过滤悬空 edge（D-8 复审 R-H1 修法）
- 节点 `remove` change → 提取 removedIds → 同步过滤 edges 中 source/target 指向已删节点的边
- 避免中间快照含 dangling edge → undo 复活悬空边

### 9. 删 isUndoRedoRef + setTimeout(100)（D-8 复审 R-M3 修法）
- applySnapshot 路径不会触发 wrappedOnNodesChange → history 不会反向递推
- snapshotsEqual 兜底防 StrictMode 双调用

## 六、踩坑记录（精华）

### A. clip-path 会裁掉 border（D-5 SVG 菱形真因）
- 决策节点用 `clipPath: 'polygon(...)'` 切菱形，**border 被一起裁掉看不见**
- 修法：改用 SVG `<polygon>` fill + stroke，4 顶点 `(50,2) (98,50) (50,98) (2,50)` 留 2px 给 strokeWidth
- 保留容器 100×100 正方形 → React Flow Handle 自然落在菱形 4 顶点

### B. html-to-image 不继承 CSS 变量（D-4 edge label SVG 显式样式真因）
- React Flow edge label 默认用 `var(--xy-edge-label-color)`，定义在 `:root`
- html-to-image 克隆 SVG 到 iframe 渲染器 → CSS 变量不继承 → fill 解析失败回退默认色
- 修法：在 `defaultEdgeOptions` 显式设 `labelStyle.fill` + `labelBgStyle.fill` 等 inline 属性

### C. fontEmbed 是 4 节点画布 5s 主因（#38 性能优化真因）
- html-to-image 默认 fontEmbed=true 会嵌入所有 @font-face 资源
- 修法：`skipFonts: true` + `cacheBust: false` + `pixelRatio 1.5` → 5s → <1s

### D. Playwright 拖拽 React Flow Handle 连线不稳（D-7 真问题）
- mouse.move + down + up 不可靠（d3-drag 内部 pointer events 时序）
- 视觉看着 PASS 但 React Flow 内部 onConnect 没真触发
- 修法：跳过 e2e 连线测试，由 useDraftHistory 单测 + 主应用 onConnect 单测兜底

### E. Playwright keydown 用 evaluate dispatchEvent 而非 page.keyboard.press
- 拖拽后焦点状态不稳，page.keyboard.press 失败
- 改 `window.dispatchEvent(new KeyboardEvent('keydown', { ... }))` 直接送到 window 监听器
- 但 e2e 撤销断言期望必须按真实栈语义（第 1 次 Ctrl+Z = 撤销改名，不是撤销添加）

### F. 用户实测"连接线没成功连接但 count 显示边 = 1"（D-7 真问题）
- Playwright mouse 模拟拖拽创建了 React Flow 内部假边但视觉没接上
- 配合 E：测试预期需要按"实际操作"而非"以为发生了什么"断言

### G. 测试预期写错被误判 D-5 bug（D-7 调试过程）
- 我以为 Case 6 "Ctrl+Z 撤销节点添加"，实际栈顶是 Case 4 改名快照
- D-5 撤销栈语义按操作时序回退正确，**测试预期错位**导致表面 FAIL
- 教训：reality check 反幻觉原则 — 别凭直觉断言，按栈顶实际状态推

### H. 我说错了 codex 建议落地代码意图（D-8 H1/H2 合修过程）
- 我以为"删 DraftNode 内 setNodes 调用"，复审 R-L1 指出"已经没有了"
- 教训：commit 修法前再核对一次实际代码，避免凭印象描述

## 八、给未来 Claude 的话

1. **嵌套式归档约定**：2026-05-13 起新功能用 `docs/规划/<功能名>/codex审查/`，**不要**塞到 `codex审查记录/阶段X/` 旧目录
2. **DraftSandbox 禁直接复用主应用画布组件**：反向断言守门覆盖 FlowCanvas / useFlowOperations / useMultiCanvas / CustomNode / useFlowHistory / useDraftAutosave / NodeDetailPanel / SaveStatus / AuthProvider / useAuth
3. **DraftSandbox 禁服务端通信**：fetch /api/、axios、ApiError、主应用 /api/ 模块 import 全禁
4. **DraftSandbox 禁实时通道**：WebSocket / EventSource / socket.io 全禁
5. **未来加 JSON/JPG/SVG 多格式导出前先评估产品红线**："游客仅图片格式" 是 §2.2 硬约束
6. **撤销栈 push snapshot 时机**：节点 add/remove/replace + 边 add/remove/replace + 拖动结束 + 改名 + edge label 改。**所有路径都走 setNodes/setEdges updater 内同步推快照**，不再用 queueMicrotask 读 ref
7. **节点删除时同步过滤悬空 edge**（避免 dangling edge 快照）
8. **applySnapshot 路径不要触发 pushSnapshot**（snapshotsEqual 已兜底，但小心 React Flow 升级后行为变化）
9. **CustomEvent 通信曾踩坑**（H2+M5）：跨组件状态联动用 React Context 不用 window event
10. **html-to-image 不继承 CSS 变量**：所有 SVG inline 样式必须显式设，否则截图渲染失败回退默认色
